# Dashboard Reading Rules — Accuracy First

> These rules prevent data misreading errors that destroy credibility. Follow them EVERY time you analyze a dashboard image.

---

## The 12 Rules

### Rule 1: Match Numbers to EXACT Labels
Read the label next to the number. "Team C: 73,071" means Team C, not Team B.
**Never** assign a number to a different label than the one displayed.

### Rule 2: Never Assume the First Row is an Aggregate
Each row has its own label. Read it. "Operations: 199" ≠ "Total: 199".

### Rule 3: Read Chart Axes Precisely
Trace data points to axis values. Don't interpolate between gridlines.
If a bar reaches between 20 and 25, say "approximately 22", not "25".

### Rule 4: Match User Categories Exactly
If the dashboard says "Unlicensed Chat Users: 2,585", write exactly that.
Don't paraphrase as "Free users" or "Non-paying users".

### Rule 5: Identify Selected Filters
Dark/black elements are selected. Grey/dimmed are not. This changes what data is shown.
Always note the active filter state.

### Rule 6: Verify Column Headers in Tables
Read left to right. "Organization | Users | Sessions/User" — don't skip or swap columns.

### Rule 7: Unit Consistency
If the dashboard shows "13K", write "13K", not "13,000".
If it shows "2,585", write "2,585", not "~2.6K".

### Rule 8: Don't Invent Numbers
Only cite numbers you can visually confirm on the dashboard image.
"Can I point to this number?" If no → don't cite it.

### Rule 9: Read the Legend
Match colors in charts to their legend labels. Red might be "Support", blue might be "IT".

### Rule 10: Context Before Numbers
Always format as: **[Label]: [Value] [Unit]**
- ✅ "Operations leads at 4.3 weekly sessions/user"
- ❌ "4.3 is the highest number"

### Rule 11: Executive Summary Spans ALL Pages
Don't write the summary after reading page 1. Read ALL pages first, then synthesize.

### Rule 12: Recommendations Must Be Actionable
Ground every recommendation in a specific data point.
- ✅ "Convert IT (Rank #1, 2.0 sessions/wk) to licensed — highest consistency"
- ❌ "Increase adoption across departments"

---

## Pre-Analysis Checklist

Run this mentally for EVERY dashboard page before writing insights:

- [ ] What filters are active on this page?
- [ ] What are all the table column headers?
- [ ] What do the chart axes show?
- [ ] Is there a legend? What does each color mean?
- [ ] What units are displayed (K, %, raw numbers)?
- [ ] Can I trace every number I plan to cite back to a visible element?
- [ ] Am I reading the correct label for each value?
